## Исследование: лендинг «Безопасно.Форум»

**Дата:** 26 ноября 2025  
**Цель:** определить технологические и UX-практики для React-лендинга форума с TailGrids, анимированным градиентом и акцентом на регистрацию.

### 1. React + Tailwind + TailGrids
- TailGrids распространяет 600+ компонентов с плагином `@tailgrids/plugin` для Tailwind, что позволяет использовать готовые шаблоны кнопок, карточек и сеток без ручной верстки и поддерживать единый визуальный стиль.[^1]
- Для интеграции TailGrids необходимо: установить npm-пакет, подключить плагин в `tailwind.config.js`, импортировать базовые стили и применять комбинацию utility-классов Tailwind и `tg-*` модификаторов. Это ускоряет создание секций «Программа», «Партнёры» и карточек спикеров и упрощает поддержку дизайн-системы.

### 2. UX акценты и CTA
- NN/g подчёркивает, что CTA должен оставаться заметным, повторяться на ключевых точках и быть сформулированным действием («Получить пропуск»), иначе конверсия падает.[^2] => Реализуем фиксированную кнопку после первого экрана + CTA внутри критических блоков.
- Контенту мероприятия помогают «живые» доказательства (кейсы, отзывы, фото) и чёткий ответ «что получу» сразу после hero. В приложении выделим блок «Что вы получите» сразу под программой и добавим логотипы организаторов для доверия.

### 3. Анимированный градиент поверх фото
- CSS-Tricks рекомендует комбинировать `background-image: linear-gradient(...) , url(...)` и выносить анимацию в `@keyframes` с плавным `background-position`/`background-size`, что создаёт «живой» фон без тяжёлых видео.[^3] => Используем фотографию `foto/00263_3 1.jpg` в `::before` или отдельном контейнере и накладываем анимированный градиент с `mix-blend-mode: overlay` для читабельности.

### 4. Доступность и медиа
- WCAG 2.1 требует текстовые альтернативы для всех значимых изображений, особенно портретов спикеров, чтобы пользователи вспомогательных технологий понимали контекст.[^4] => Добавляем `alt` с должностью/именем; декоративные паттерны помечаем пустым `alt`.

### 5. Контентная структура
- Для event-лендингов рекомендуется следовать последовательности: герой (кто/что/когда/где) → аудитория/выгода → программа → спикеры → партнёры → регистрация. Такой поток поддерживает «information scent» и уменьшает когнитивную нагрузку.[^2]
- Пакеты партнёрства вынесем в отдельный раздел с CTA «Стать партнёром», а коммерческое предложение можно позже оформить отдельной страницей/документом.

### 6. Следующие шаги
1. Развернуть React (Vite) + Tailwind + TailGrids и подготовить общие layout-компоненты.
2. Сконфигурировать глобальный градиент-фон с анимацией и обеспечить лёгкую замену фонового фото.
3. Реализовать секции по структуре из п.5 с карточками спикеров/гостей и единым CTA.
4. Добавить доступные подписи, шаблон партнёрских пакетов и секцию регистрации с кнопкой, ведущей к боту.
